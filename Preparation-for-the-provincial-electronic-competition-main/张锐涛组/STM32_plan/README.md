# STM32F103C8T6 4x4矩阵键盘 + AD9851 2ASK发送系统

基于 STM32F103C8T6 的 2ASK（幅移键控）数字信号发送系统，支持 4x4 矩阵键盘输入、OLED 显示和 AD9851 DDS 载波调制。

## 1. 快速开始

### 1.1 编译

用 Keil MDK v5 打开 `USER/VirtualCOMPort.uvprojx`，点击 Rebuild 编译下载。

### 1.2 模式切换

在 `USER/main.c` 修改第 26 行的 `WORK_MODE` 宏：

```c
#define WORK_MODE  0   // 键盘测试模式
#define WORK_MODE  1   // 2ASK 发送模式（默认）
```

## 2. 工作模式

### 模式 0：键盘测试

OLED 实时显示每个按键的：
- **第 1 行**：按键字符 + 十六进制键值
- **第 2 行**：所在行列对应的 MCU 引脚名称
- **第 3 行**：累计按键次数

同时通过串口（9600 baud）输出调试信息。

### 模式 1：2ASK 发送

键盘输入 → STM32 编码 → AD9851 产生 11.5kHz 2ASK 调制信号。

**操作方法：**

| 按键 | 功能 |
|------|------|
| `0-9`, `B`, `*` | 数据输入，追加到发送缓冲区 |
| `A` | 将缓冲区数据以 2ASK 格式发送 |
| `C` | 清空发送缓冲区 |
| `D` | 连续载波（CW）模式开关：持续输出未调制 11.5kHz 正弦波 |
| `#` | 调试模式开关：循环发送 `0x55` 2ASK 信号 |

> CW 模式和调试模式互斥，开启一个会自动关闭另一个。

**OLED 显示：**

| 行 | 内容 |
|----|------|
| 第 1 行 | "2ASK TX Ready" |
| 第 2 行 | 状态提示 / `A=Send C=Clr #=Dbg` / `CW: 11.5kHz ON` / `Debug: 0x55 loop` |
| 第 3 行 | 发送缓冲区内容（实时更新） |

**调试与测试功能：**

| 模式 | 触发 | 输出内容 | 用途 |
|------|------|----------|------|
| 连续载波（CW） | 按 `D` | 纯净 11.5kHz 正弦波 | 验证 DDS 输出、测频率/幅度 |
| 调试发送 | 按 `#` | 循环 2ASK 信号（`0x55` = `01010101`） | 示波器观察完整 2ASK 波形 |
| 正常发送 | 按 `A` | 缓冲区数据（含帧头/校验） | 实际通信 |

## 3. 硬件引脚分配

**MCU：STM32F103C8T6（LQFP48）**

### 3.1 矩阵键盘（4x4）

| 信号 | 引脚 | 说明 |
|------|------|------|
| ROW0 | **PB10** | 行输出（推挽） |
| ROW1 | **PB9** | 行输出（推挽） |
| ROW2 | **PB1** | 行输出（推挽） |
| ROW3 | **PB11** | 行输出（推挽） |
| COL0 | **PA0** | 列输入（上拉） |
| COL1 | **PA1** | 列输入（上拉） |
| COL2 | **PA2** | 列输入（上拉） |
| COL3 | **PA3** | 列输入（上拉） |

### 3.2 AD9851 DDS

| 信号 | 引脚 | 说明 |
|------|------|------|
| D7 (DATA) | **PB5** | 串行数据线 |
| RST | **PB6** | 复位 |
| FQ_UD | **PB7** | 频率更新 |
| CLK | **PB8** | 串行时钟 |

晶振：30MHz，6 倍频后系统时钟 180MHz。

### 3.3 OLED 显示屏（SSD1306，I2C）

| 信号 | 引脚 |
|------|------|
| SCL | **PB12** |
| SDA | **PB13** |

I2C 地址：`0x3C`（7 位），自动探测 `0x3C` / `0x3D`。

### 3.4 串口调试

| 信号 | 引脚 |
|------|------|
| TX | **PA9** |
| RX | **PA10** |

波特率：9600，用于 printf 调试输出。

## 4. 物理按键布局

由于 PCB 行列线序与扫描代码相反，实际按键布局如下：

```
+-----+-----+-----+-----+
|  D  |  #  |  0  |  *  |   ROW0 (PB10)
+-----+-----+-----+-----+
|  C  |  9  |  8  |  7  |   ROW1 (PB9)
+-----+-----+-----+-----+
|  B  |  6  |  5  |  4  |   ROW2 (PB1)
+-----+-----+-----+-----+
|  A  |  3  |  2  |  1  |   ROW3 (PB11)
+-----+-----+-----+-----+
  COL0   COL1  COL2  COL3
 (PA0)  (PA1) (PA2) (PA3)
```

> 若实际 PCB 丝印与此不同，可修改 `SYSTEM/KEYBOARD/keyboard.c` 和 `USER/main.c` 中的 `key_map[4][4]` 数组校正。

## 5. 2ASK 编码格式

| 参数 | 值 |
|------|-----|
| 调制方式 | OOK（通断键控）/ 2ASK |
| 载波频率 | 11.5 kHz |
| 波特率 | 100 bps |
| 位时长 | 10 ms |
| 位格式 | 1 起始位(0) + 8 数据位(LSB) + 1 停止位(1) |
| 数据帧 | `[0xAA] [长度] [数据...] [校验和]` |
| 最大数据长度 | 16 字节 |

AD9851 通过 `ad9851_on` / `ad9851_off` 控制位实现载波的通断：
- **bit = 1**：输出 11.5kHz 正弦波（DAC 上电）
- **bit = 0**：输出关闭（DAC 下电）

## 7. 常见问题

**Q：编译时出现 `Undefined symbol ...` 链接错误？**  
A：检查缺失函数所在的 `.c` 文件是否已添加到 Keil 工程的对应 Group 中。

**Q：按键显示字符与实际丝印不符？**  
A：修改 `keyboard.c` 和 `main.c` 中的 `key_map[4][4]` 数组，使键值映射匹配实际 PCB 走线。

**Q：OLED 不显示？**  
A：检查 PB12/PB13 上拉电阻（4.7kΩ~10kΩ），确认 I2C 地址（`0x3C` / `0x3D`）。串口会输出 OLED 初始化调试信息。

**Q：AD9851 无输出？**  
A：检查 30MHz 晶振是否起振，确认 PB5-PB8 引脚连接正确，示波器测量输出端应有 11.5kHz 正弦波（发送数据时）或直流（空闲时）。

**Q：串口无输出？**  
A：确认 PA9/PA10 连接正确，波特率 9600，8N1。USB 转串口模块的 TX 接 PA10，RX 接 PA9。
